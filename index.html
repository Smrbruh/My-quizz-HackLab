Section 01

Authentication Security

Master authentication vulnerabilities, attack techniques, exploitation methods, and defense mechanisms used in modern web applications.

⚡ Quick Reference — Authentication
Brute-Force Indicators
  • Multiple failed login attempts
  • IP-based rate limiting absent
  • No account lockout policy
  • Predictable lockout logic
  • Status code differences
  • Response time variations
MFA Weaknesses
  • OTP reuse allowed
  • No brute-force protection on OTP
  • Step-skip possible (direct URL)
  • Long OTP validity window
  • Insecure backup codes
  • SIM-swap susceptibility
OAuth Weaknesses
  • Missing state parameter
  • Open redirect abuse
  • Scope escalation
  • Token leakage via Referer
  • Implicit grant misuse
  • Account linking flaws
Password Reset Flaws
  • Predictable reset tokens
  • Token reuse after use
  • No token expiry
  • Host header injection
  • User enumeration via response
  • Token in URL (logged by proxy)
